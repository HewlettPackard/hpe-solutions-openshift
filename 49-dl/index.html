# Introduction

This document describes the deployment of a high available and secure Red Hat® OpenShift® Container Platform 4.9 on HPE Proliant DL380 and DL360 Gen10 Servers. It is intended to be used in conjunction with Ansible playbooks and python scripts found at https://github.com/HewlettPackard/hpe-solutions-openshift (opens new window). This document was created using OpenShift Container Platform 4.9 and the documents for that version are available at https://docs.openshift.com/container-platform/4.9/welcome/index.html (opens new window). Hewlett Packard Enterprise strives to make regular updates to this solution with the goal of validating against the latest available version of Red Hat OpenShift Container Platform. As a result, references in the document will generally refer to OpenShift Container Platform 4.9 and will mention specific sub-versions in strategic locations to note the version that was tested.

The Red Hat OpenShift Container Platform 4.9 deployment on HPE ProlianDL360 and DL380 Gen 10 servers consists of the following:

- Three (3) OpenShift Container Platform master nodes Three (3) HPE ProLiant DL380 Gen10 servers are used for the worker nodes (out of which one node will be temporarily used as the OCP bootstrap node). Later the bootstrap node will be reconfigured as an additional compute node and storage node.

Red Hat OpenShift Container Storage Red Hat® OpenShift® Data Foundation—previously Red Hat OpenShift Container Storage—is software-defined storage for containers. Engineered as the data and storage services platform for Red Hat OpenShift.

- Persistent volume leverages Red Hat OpenShift Data Foundation4.8 (Internal or External) or HPE Storage systems such as HPE Nimble and HPE Primera provide persistent storage for containers and registry as well as data management.

OpenShift Data Foundation (ODF) will be installed on the three worker nodes internally. Each ODF node is equipped with a single SSD drive to be used as the ODF storage device. Optionally, the customer can configure the ODF in external storage mode. For this purpose, three more HPE ProLiant DL380 Gen10 will be required to form a Red Hat Ceph storageexternal storage cluster. Based on use the use HPE storage system can be used for the External storage mode.

It is recommended this document should be reviewed in its entirety and the installation user should understand all prerequisites and procedure before installation. It is also recommended that the installation user review the OpenShift Container Platform 4.9 installation process as described by Red Hat.

  • Due to the ephemeral nature of the containers, protection of persistent data associated with containers is a crucial task. The pods which require persistent data use persistent volume with disks from the HPE Storage system.
